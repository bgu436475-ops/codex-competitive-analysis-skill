# Cursor A/B 对比：普通提示词 vs. Evidence-First Skill

## 实验设置

- **固定输入：**两次运行均使用同一份 [`prompt.md`](prompt.md)，研究日期均为 2026-07-14。
- **预期控制变量：**两次运行获得相同的研究任务与预期可用工具；实验组只额外获得完成后的 `$competitive-analysis` Skill。运行记录没有完整保存模型版本、工具清单和每次工具调用元数据，因此“相同工具”是实验设置意图，不能视为已证明两次执行环境完全一致。
- **产物完整性：**[`baseline-report.md`](baseline-report.md) 是不加载 Skill 的独立原始最终输出，未经回写。最终 [`skill-report.md`](skill-report.md) 来自全新的 prompt+Skill-only 独立运行，随后按 Skill 合约做了结构审计与修订；最终文件 SHA-256 为 `5e1812b713723c41040b747dece3f66f87d3b8333fcb42b854c0321886c86f3d`。
- **评价范围：**只评价报告中可见的证据与决策支持，不把本次链接可达性检查当作报告当时已执行的研究，也不对所有产品事实做独立真伪审计。

这是一组单次流程演示，用于展示 Skill 及其合约审计对最终交付的影响。由于实验组包含生成后的结构审计修订，它不是盲测基准，不能把差异完全归因于首次模型输出，也不能证明统计显著性。

## 评分方法

六个维度各 0–5 分，总分 30。通用含义为：0 = 缺失；1 = 仅提及；2 = 局部执行；3 = 基本满足；4 = 强且大体一致；5 = 系统化、可审计且几乎无关键缺口。为减少事后调整，每个维度还使用以下满分锚点：

|维度|5 分锚点|
|---|---|
|来源质量与新鲜度|关键主张有新鲜的精确来源，并兼顾直接官方证据与独立佐证；日期、归因完整。|
|主张可核验性|每个重要主张可原子化映射到精确链接、访问日、等级和限制。|
|一手体验|多个核心路径被实际执行，环境、入口、步骤、结果和账户/地区限制均可复现。|
|事实/推断分离|所有重要事实、观察、推断和建议均明确分类，叙述与登记一致。|
|分析深度|完整覆盖任务维度，并以交叉证据、客户证据或情景分析检验关键判断。|
|决策价值|建议有优先级、决策目标、证据依赖、影响、置信度、投入和时间范围。|

|维度|普通提示词|使用 Skill|差值|可复核依据|

|来源质量与新鲜度|4|4|0|普通报告在[竞争对比](baseline-report.md#竞争对比)中链接 13 个不同页面，主要是官方源，并用 AP 补充交易信息；但来源未逐条标访问日，多项采用数据仍是厂商自报。Skill 报告在[来源登记表](skill-report.md#来源登记表)中记录 15 条官方直接来源、访问日和等级；仍缺独立二手佐证，因此同为 4 分而非满分。|
|主张可核验性|3|5|+2|普通报告有行内链接，但一个段落或链接常承载功能、价格、采用与评价等多种命题，读者难以逐句审计。最终 Skill 报告的[声明登记表](skill-report.md#声明登记表)列出 32 行经合约审计的主张，给出完整命题、类型、等级、精确来源、日期、置信度和备注，并在正文复用 ID。|
|一手体验|0|2|+2|普通报告的[研究限制](baseline-report.md#研究限制)说明只使用公开网页，没有执行或记录产品路径。Skill 报告的[第一手体验日志](skill-report.md#第一手体验日志)记录四个公开入口、工具返回和内部路径状态；但四项内部路径均为 `Not firsthand-verified`，没有走通真实产品流程，所以只得 2 分。|
|事实/推断分离|2|5|+3|普通报告会用“可能”“尚未证明”“推断”等词提示不确定性，但[结论摘要](baseline-report.md#结论摘要)和[竞争对比](baseline-report.md#竞争对比)仍把来源事实、优势判断和战略含义混在同段。Skill 报告在[声明登记表](skill-report.md#声明登记表)中逐条使用 FACT、OBSERVATION、INFERENCE、RECOMMENDATION，并在正文引用对应 ID。|
|分析深度|4|4|0|普通报告的[竞争对比](baseline-report.md#竞争对比)与[市场空白和建议](baseline-report.md#市场空白与优先建议)覆盖四个产品、定价、采用、优劣势和三项方向，叙述更丰富，但若干判断缺少可审计前提。Skill 报告的[证据支持的发现](skill-report.md#证据支持的发现)、[竞争比较](skill-report.md#竞争比较)与机会表更系统，却主要依赖厂商材料，未做客户访谈、统一基准或真实产品试用。|
|决策价值|4|4|0|普通报告的[三项建议](baseline-report.md#市场空白与优先建议)有明确顺序和业务指标，但没有显式给出置信度、投入与证据依赖。Skill 报告的[优先建议](skill-report.md#优先建议)给出顺序、服务决策与证据依赖，但简表没有逐项列出影响、置信度、投入和时间范围，因此同为 4 分。|
|**总分**|**17/30**|**24/30**|**+7**|算术合计；不是统计估计。|

## 结论

本次演示显示，Skill 的主要增量不是“写得更深”，而是把证据约束变成可检查的交付结构：普通报告引用 13 个不同页面，最终 Skill 报告登记 15 条来源和 32 条主张；两者在来源质量、分析深度和决策价值上同分。Skill 报告通过类型、等级、日期、置信度和建议依赖，让读者更容易复核事实并定位推断。

代价同样可见。机械统计显示，普通报告全文件为 3,422 个 Markdown 字符，最终 Skill 报告为 10,991 个，约为 3.21 倍；Skill 报告全文件含 1,617 个汉字（按 `[一-龥]` 计数）。建立登记表、拆分命题、执行入口检查和合约审计会增加研究时间与阅读负担。若任务只是快速扫描，这种格式可能过重；涉及产品路线、采购或高风险判断时，可追溯性更有价值。

剩余缺口没有被格式消除：四个公开入口虽然留下实际返回记录，产品内部路径仍全部未验证；采用与成效数据仍以厂商自报为主；没有客户访谈、同一代码库基准、合同折扣或留存数据。最终报告经过合约审计修订，因此评分反映的是“Skill 加审计流程”的最终交付，而不是未干预的首次输出。

因此，合理结论是：**在这次流程演示中，Skill 加合约审计明显提升了可审计性，但没有证明来源更独立、分析更深、决策价值更高或结论更正确。** 模型随机性、页面时点、工具可用性和审计修订都可能改变结果；要比较 Skill 对首次输出的稳定效果，应在多个产品、多个独立运行和盲评条件下重复实验。
